## เจตนา

เอาชุดเงื่อนไขแยกกันที่แต่ละตัวคุ้มกันผลลัพธ์ *เดียวกัน* มาพับรวมเป็นเงื่อนไขผสมเดียว แล้วสกัดเงื่อนไขนั้นออกมาเป็นฟังก์ชันที่มีชื่อ การตรวจสอบสามตัวที่กระจัดกระจายซึ่งมี `return 0` เหมือนกัน กลายเป็น `if (isNotEligible(employee)) return 0` เพียงตัวเดียว และเหตุผลของผลลัพธ์ก็ถูกจับใจความไว้ในชื่อ

## กลิ่นโค้ด

นี่คือวิธีแก้สำหรับ **ลำดับของเงื่อนไขที่มีผลลัพธ์เหมือนกัน** การตรวจสอบแต่ละตัวลำพังดูไร้เดียงสา แต่เมื่ออยู่ด้วยกัน มันซ่อนแนวคิดเดียวไว้หลังท่อร้อยสายสามบรรทัด ผู้อ่านต้องสังเกตให้ได้ว่าทั้งสามตัวให้ค่าเหมือนกันก่อนจะเข้าใจว่าจริง ๆ แล้วมันคือคำถามเดียว — *คนนี้ขาดคุณสมบัติใช่ไหม?* การรวมเข้าด้วยกันทำให้คำถามเดียวนั้นชัดแจ้ง และตัวช่วยที่มีชื่อก็บอกผู้อ่านคนถัดไปว่า *ทำไม* โดยไม่ต้องให้เขาวิศวกรรมย้อนกลับ

## Before → After

จำนวนเงินสวัสดิการคนพิการ return ศูนย์สำหรับเงื่อนไขที่ตัดสิทธิหลายข้อ ก่อนหน้านี้ ตัวตัดสิทธิแต่ละตัวเป็น `if` ของตัวเอง หลังจากนั้น มันเป็นการตรวจสอบที่มีชื่อเพียงตัวเดียว

## กลไก

1. ยืนยันว่าเงื่อนไขเหล่านั้นมีผลลัพธ์ร่วมกันจริง ๆ และไม่มีผลข้างเคียง (side effect) ระหว่างกัน หากการตรวจสอบหนึ่งเปลี่ยนสถานะที่ตัวถัดไปพึ่งพา อย่ารวม
2. รวมการทดสอบด้วย `||` เมื่อทั้งหมดนำไปสู่สาขาเดียวกัน (หรือ `&&` เมื่อโครงสร้างเป็นการตรวจสอบซ้อนกันที่ต้องผ่านทั้งหมด) วางเงื่อนไขที่รวมแล้วไว้ติดกับผลลัพธ์ร่วม
3. **รันเทสต์ของคุณ** พฤติกรรมต้องตรงกับโซ่ของ `if` แยกกันทุกประการ รวมถึงลำดับการลัดวงจร (short-circuit)
4. ใช้ **Extract Function** กับเงื่อนไขที่รวมแล้ว ตั้งชื่อตามคำถามที่มันตอบ (`isNotEligible`)
5. รันเทสต์ของคุณอีกครั้ง

## เมื่อไรควรใช้ / ข้อแลกเปลี่ยน

หยิบ Consolidate Conditional Expression มาใช้เมื่อใดก็ตามที่คุณเห็นการตรวจสอบติดต่อกันซึ่งทุกตัวนำไปสู่ผลลัพธ์เดียวกัน มันมีค่าที่สุดในฐานะท่าตั้งต้น: เมื่อการตรวจสอบกลายเป็นฟังก์ชันที่มีชื่อตัวเดียว คุณสามารถนำฟังก์ชันนั้นกลับมาใช้ใหม่ และมันมักเปิดทางให้ **Replace Nested Conditional with Guard Clauses**

อย่ารวมเมื่อการตรวจสอบเป็นการตัดสินใจที่เป็นอิสระต่อกันจริง ๆ ซึ่งบังเอิญมีค่าร่วมกันในวันนี้แต่อาจแยกทางกันในวันพรุ่งนี้ — การรวมมันจะทำให้กฎที่ไม่เกี่ยวข้องกันมาผูกติดกัน และอย่ารวมข้ามผลข้างเคียงเด็ดขาด การลัดวงจรของนิพจน์ที่รวมแล้วอาจข้ามงานที่โซ่เดิมทำไว้
