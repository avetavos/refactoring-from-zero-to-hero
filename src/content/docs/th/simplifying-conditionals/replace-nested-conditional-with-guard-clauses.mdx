## เจตนา

เอาฟังก์ชันที่งานจริง ๆ ฝังอยู่ภายในบันไดของบล็อก `if/else` แล้วดึงแต่ละกรณีพิเศษออกมาไว้ด้านบนในรูปของ **guard clause** — การตรวจสอบที่ return ออกไปทันที เมื่อทุกกรณียกเว้นออกไปก่อนแล้ว เส้นทางปกติก็เหลืออยู่ที่ระดับการย่อหน้าฐาน ซึ่งมันอ่านได้เหมือนเป็นจุดประสงค์ที่แท้จริงของฟังก์ชัน

## กลิ่นโค้ด

นี่คือวิธีแก้สำหรับ **เงื่อนไขที่ซ้อนกัน** ซึ่งทำให้หาเส้นทางหลัก (happy path) ได้ยาก เมื่อบล็อก `if/else` ซ้อนกันลึกสามหรือสี่ชั้น บรรทัดที่คุณสนใจจริง ๆ จะอยู่ไกลไปทางขวา และคุณต้องจดจำทุกเงื่อนไขที่ห่อหุ้มอยู่ไว้ในหัวเพื่อรู้ว่ามันรันเมื่อไร guard clause พลิกการเน้น: การ return ออกไปก่อนแต่ละครั้งบอกว่า *"กรณีแปลก ๆ นี้จัดการเรียบร้อยแล้ว ตอนนี้ลืมมันไปได้"* ดังนั้นเมื่อคุณมาถึงด้านล่าง ก็เหลือเพียงสถานการณ์หลักเท่านั้น

## Before → After

ฟังก์ชันเงินเดือน return จำนวนเงินที่ต่างกันสำหรับพนักงานที่เสียชีวิต ออกจากงาน และเกษียณ โดยมีเงินเดือนปกติเป็นค่าดีฟอลต์ ก่อนหน้านี้ กรณีต่าง ๆ ซ้อนกัน หลังจากนั้น แต่ละกรณีเป็น guard

## กลไก

1. เลือกเงื่อนไขชั้นนอกสุดที่จัดการกรณีพิเศษ แทนสาขาของมันด้วยการ return ออกไปก่อน แล้วยกมันขึ้นไปไว้บนสุดของฟังก์ชัน
2. **รันเทสต์ของคุณ** ผลลัพธ์สำหรับกรณีนั้นต้องเหมือนเดิม
3. ขยับไปยังเงื่อนไขชั้นนอกถัดไปและทำแบบเดียวกัน ลบ `else` ออกได้เพราะกรณีด้านบน return ไปแล้ว
4. ทำต่อไปจนทุกกรณีพิเศษเป็น guard และเหลือเพียงเส้นทางหลักที่ด้านล่าง แบบไม่ย่อหน้า
5. หากเงื่อนไขของ guard อ่านขัดเขินเมื่อกลับด้าน ลองพิจารณาสกัดมันด้วย **Decompose Conditional** เพื่อให้การ return ออกไปก่อนยังอ่านได้สะอาด

## เมื่อไรควรใช้ / ข้อแลกเปลี่ยน

หยิบ guard clause มาใช้เมื่อการซ้อนกันบดบังการไหลปกติของฟังก์ชัน โดยเฉพาะเมื่อสาขาหลายตัวเป็น *ข้อยกเว้น* มากกว่าเป็นทางเลือกที่เท่าเทียมกัน การ refactor นี้โดดเด่นเมื่อกรณีต่าง ๆ เป็นแบบ "จัดการแล้วจากไป" จริง ๆ — การตรวจสอบข้อผิดพลาด ข้อมูลที่หายไป สถานะขอบเขต

มันเหมาะสมน้อยลงเมื่อสาขาเป็นทางเลือกคู่ขนานที่มีน้ำหนักเท่ากันจริง ๆ การยัดสิ่งเหล่านั้นให้เป็น guard อาจสื่อถึงลำดับความสำคัญที่ไม่มีอยู่จริง บางทีมยังชอบจุดออกเพียงจุดเดียวต่อฟังก์ชัน — หากทีมของคุณเป็นเช่นนั้น ก็ชั่งน้ำหนักธรรมเนียมนั้นกับประโยชน์ด้านความอ่านง่าย ในโค้ดสมัยใหม่ส่วนใหญ่ เวอร์ชันที่ราบเรียบกว่าชนะ
