## เจตนา

ผู้เรียกใช้ถือออบเจกต์อยู่แล้ว แล้วก็ขุดเอาฟิลด์สองสามตัวออกมาแค่เพื่อส่งมันเข้าฟังก์ชันทีละตัว ให้ส่ง **ทั้งออบเจกต์** ข้ามไปแทน รายการอาร์กิวเมนต์จะหดลง ฟังก์ชันสามารถเอื้อมไปหยิบฟิลด์ใดก็ได้ที่มันต้องการ และถ้าออบเจกต์งอกฟิลด์ใหม่ที่เกี่ยวข้องในภายหลัง ลายเซ็นก็ไม่ต้องเปลี่ยน

## กลิ่นไม่ดี

ที่จุดเรียกใช้คุณจะเห็นชุดของการเรียก accessor ที่ป้อนตรงเข้าไปในอีกการเรียกหนึ่ง: `plot(point.x, point.y)`, `fits(range.low, range.high)` ค่าที่ถูกแกะออกมาทั้งหมดมาจากออบเจกต์เดียวกันและเดินทางไปด้วยกัน รายการพารามิเตอร์ของฟังก์ชันสะท้อนเนื้อในของออบเจกต์ ทั้งสองจึงถูกผูกติดกันอย่างเงียบ ๆ ในขณะที่แกล้งทำเป็นอิสระต่อกัน

## ก่อน → หลัง

ค่าที่อ่านได้ถูกตรวจกับ `range` ของอุณหภูมิ ผู้เรียกใช้ดึง `low` และ `high` ออกจาก range เพื่อส่งเข้าไป แทนที่จะทำอย่างนั้น ให้ส่ง range เองและปล่อยให้การตรวจถามมันโดยตรง

## กลไก

1. เพิ่มพารามิเตอร์ใหม่สำหรับทั้งออบเจกต์ โดยปล่อยพารามิเตอร์ฟิลด์ที่มีอยู่ไว้ก่อน
2. ภายในฟังก์ชัน แทนที่การใช้ฟิลด์ที่ถูกแกะแต่ละจุดด้วยการอ่านจากทั้งออบเจกต์
3. อัปเดตผู้เรียกใช้แต่ละจุดให้ส่งออบเจกต์และเลิกส่งฟิลด์แต่ละตัว **รันเทสต์** หลังจากแต่ละจุด
4. เมื่อแปลงผู้เรียกใช้ทุกจุดแล้ว ให้ลบพารามิเตอร์ฟิลด์ที่ตอนนี้ไม่ถูกใช้แล้วทิ้ง
5. มองหาตรรกะในผู้เรียกใช้ที่จัดการแค่ฟิลด์เหล่านั้น — มันอาจควรอยู่ภายในฟังก์ชัน หรือเป็นเมท็อดบนตัวออบเจกต์เอง

## เมื่อไรควรใช้ / ข้อแลกเปลี่ยน

หยิบท่านี้มาใช้เมื่อฟิลด์ที่ถูกแกะออกมาทั้งหมดเป็นของออบเจกต์เดียวกัน เมื่อหลายฟังก์ชันดึงกลุ่มฟิลด์เดียวกันออกมา หรือเมื่อฟังก์ชันน่าจะได้ประโยชน์จากการเข้าถึงออบเจกต์มากกว่าที่ได้รับอยู่ในปัจจุบัน มันมักเผยให้เห็น **Feature Envy** — โค้ดที่ไปยุ่งกับข้อมูลของออบเจกต์อื่น — ซึ่งคุณก็สามารถย้ายไปไว้บนออบเจกต์นั้นเป็นเมท็อดได้

ข้อแลกเปลี่ยนคือ dependency ที่แน่นขึ้น: ตอนนี้ฟังก์ชันรู้จัก type ของออบเจกต์ ไม่ใช่แค่ตัวเลขธรรมดาสองตัว ถ้าผู้ถูกเรียกควรไม่รู้จัก type นั้น — เพื่อนำกลับมาใช้ในบริบทที่ไม่เกี่ยวข้องกัน หรือเพื่อเลี่ยง circular dependency — ให้คงพารามิเตอร์แบบหลวมไว้ และถ้าผู้เรียกใช้ยังไม่มีทั้งออบเจกต์อยู่ในมือ การประกอบมันขึ้นมาแค่เพื่อจะเรียกถือเป็นการถอยหลังก้าวหนึ่ง
