## เจตนา

นำฟังก์ชันที่ใช้อาร์กิวเมนต์แบบ boolean เพื่อเลือกระหว่างสองพฤติกรรมที่แตกต่างกัน แล้วแทนที่ด้วยสองฟังก์ชันแยกกัน — หนึ่งฟังก์ชันต่อหนึ่งพฤติกรรม — แต่ละตัวมีชื่อที่บอกว่ามันทำอะไร ผู้เรียกใช้เลิกส่ง `true` ที่เข้ารหัสยาก แล้วเริ่มเรียกฟังก์ชันที่ชื่อของมันคือคำตอบ

## กลิ่นไม่ดี

flag argument ซ่อนเจตนาไว้ที่จุดเรียกใช้ เมื่อคุณอ่าน `sendEmail(order, true)` ตัว `true` ไม่ได้บอกอะไรคุณเลย — คุณต้องเปิดฟังก์ชันเพื่อจะรู้ว่า `true` หมายถึง "ส่งด่วน" ในขณะที่ `false` หมายถึง "เข้าคิวไว้ทีหลัง" boolean กำลังทำงานที่ชื่อฟังก์ชันควรทำ มันยังมักทำให้เนื้อในแตกแขนงด้วย: `if (urgent) ... else ...` ที่ยัดสองกระแสงานที่ไม่เกี่ยวข้องกันเข้ามาในฟังก์ชันเดียว ทำให้ทั้งสองอ่านไม่สะอาดและทั้งคู่ต้องถูกเทสต์ผ่านประตูเดียวกัน

## ก่อน → หลัง

ฟังก์ชันแจ้งเตือนที่ boolean เลือกระหว่างเส้นทางด่วนและเส้นทางปกติ หลังจากนั้น สองฟังก์ชันที่มีชื่อ แต่ละตัวมุ่งเน้น

## กลไก

1. ยืนยันว่า flag เลือกระหว่าง *พฤติกรรม* จริง ๆ ไม่ใช่แค่ค่า ถ้ามันแค่สลับตัวเลขหรือป้ายกำกับ Parameterize Function อาจเหมาะกว่าการแยก
2. สร้างฟังก์ชันใหม่สำหรับแขนงหนึ่งของ flag ตั้งชื่อให้จับเจตนาของแขนงนั้น อย่าง `notifyUrgent`
3. คัดลอกตรรกะของแขนงที่เกี่ยวข้องเข้าไปในฟังก์ชันใหม่ แล้วทิ้งแขนงที่ตายแล้ว **รันเทสต์ของคุณ**
4. หาผู้เรียกใช้ทุกตัวที่ส่ง flag ด้วยค่านั้น แล้วเปลี่ยนเส้นทางไปยังฟังก์ชันใหม่ พร้อมลบอาร์กิวเมนต์ boolean ออก
5. ทำซ้ำขั้นที่ 2 ถึง 4 สำหรับอีกแขนงหนึ่ง
6. **รันเทสต์ของคุณ** หลังการเปลี่ยนเส้นทางแต่ละครั้ง เพื่อให้ความล้มเหลวชี้ไปยังผู้เรียกใช้จุดเดียว
7. เมื่อไม่มีผู้เรียกใช้ตัวใดส่ง flag อีกแล้ว ให้ลบฟังก์ชันเดิมทิ้ง

## เมื่อไรควรใช้ / ข้อแลกเปลี่ยน

แยก flag argument ออกเมื่อใดก็ตามที่พารามิเตอร์ boolean เปลี่ยน *สิ่งที่ฟังก์ชันทำ* เมื่อใดที่ `true` หรือ `false` แบบ literal ที่จุดเรียกใช้อ่านไม่รู้เรื่อง หรือเมื่อใดที่เนื้อในแตกออกเป็นสองกระแสงานที่แทบไม่มีอะไรร่วมกัน สองฟังก์ชันที่มีชื่อทำให้การเรียกใช้แต่ละครั้งอธิบายตัวเองได้ และให้แต่ละเส้นทางถูกเทสต์ได้โดยตรง

ต้นทุนคือมีฟังก์ชันใน namespace มากขึ้น และโค้ดเตรียมการที่ใช้ร่วมกันจริง ๆ บางส่วนอาจไปอยู่ในสองที่ — ดึงส่วนที่ใช้ร่วมกันนั้นออกมาเป็น helper ส่วนตัวเพื่อไม่ให้คุณทำซ้ำมัน จงใช้สามัญสำนึกด้วย: flag ที่แค่พลิกข้อมูลการตั้งค่า โดยไม่มีการแตกแขนงเชิงพฤติกรรม คงไว้ได้ไม่เป็นไร กลิ่นที่ว่านี้เจาะจงคือ boolean ที่ *ตัดสินใจเลือกระหว่างพฤติกรรม*
