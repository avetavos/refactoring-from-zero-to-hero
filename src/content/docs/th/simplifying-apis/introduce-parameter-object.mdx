## เจตนา

มองหากลุ่มอาร์กิวเมนต์ที่ปรากฏด้วยกันอยู่เรื่อย ๆ ในหลายฟังก์ชัน แล้วแทนที่มันด้วยออบเจกต์เดียวที่เก็บมันไว้ทั้งหมด กลุ่มนั้นได้ชื่อ ความสัมพันธ์ระหว่างค่าต่าง ๆ ชัดเจนขึ้น และทุกฟังก์ชันที่เคยรับอาร์กิวเมนต์กระจัดกระจาย ตอนนี้รับพารามิเตอร์ที่เรียบร้อยเพียงตัวเดียว

## กลิ่นไม่ดี

นี่รักษา **Long Parameter List** และญาติของมัน **Data Clump** — ค่าสองหรือสามตัวเดิมที่ถูกส่งเรียงข้างกัน ในลำดับเดิม ให้ฟังก์ชันแล้วฟังก์ชันเล่า `start` และ `end` ที่จริง ๆ แล้วเป็น *ช่วง* วันที่ `latitude` และ `longitude` ที่จริง ๆ แล้วเป็น *ตำแหน่ง* เมื่อค่าต่าง ๆ เดินทางกันเป็นฝูง ฝูงนั้นคือแนวคิดที่โค้ดยังไม่ได้ตั้งชื่อ ที่แย่กว่านั้นคือ รายการเปล่า ๆ ที่ยาว ยังเชื้อเชิญความผิดพลาดคลาสสิกในการส่งอาร์กิวเมนต์ในลำดับที่ผิด

## ก่อน → หลัง

การจองเที่ยวบินที่ลากอาร์กิวเมนต์สี่ตัวที่เกี่ยวข้องกันอย่างหลวม ๆ ไปไหนมาไหน หลังจากนั้น มันกลายเป็นออบเจกต์ `Trip` เดียว

## กลไก

1. ระบุกลุ่ม — อาร์กิวเมนต์ที่ปรากฏด้วยกันเสมอ เลือกชื่อให้กับแนวคิดที่มันแทน
2. สร้างโครงสร้างใหม่ (คลาส, struct, record หรือ interface) ที่มีฟิลด์สำหรับสมาชิกแต่ละตัวของกลุ่ม
3. เพิ่มออบเจกต์ใหม่เป็นพารามิเตอร์ให้ฟังก์ชันเป้าหมายตัวหนึ่ง โดยตอนแรกวางไว้ข้าง ๆ อาร์กิวเมนต์เดิม **รันเทสต์ของคุณ**
4. ภายในเนื้อใน ให้เริ่มอ่านจากฟิลด์ของออบเจกต์แทนพารามิเตอร์ที่กระจัดกระจาย ทีละฟิลด์ โดยเทสต์ไปเรื่อย ๆ ขณะทำ
5. เมื่อเนื้อในอ่านจากออบเจกต์เพียงอย่างเดียวแล้ว ให้ลบพารามิเตอร์ที่กระจัดกระจายซึ่งไม่ได้ใช้แล้วออกจากลายเซ็น จากนั้นอัปเดตผู้เรียกใช้ของฟังก์ชันนั้นให้สร้างและส่งออบเจกต์
6. ทำซ้ำกับฟังก์ชันอื่น ๆ ที่ใช้กลุ่มเดียวกัน เพื่อให้ทั้งหมดรับออบเจกต์
7. **รันเทสต์ของคุณ** หลังแต่ละฟังก์ชัน คอยจับตาดูพฤติกรรมใหม่ที่ผุดขึ้นมา: เมื่อข้อมูลอยู่ด้วยกันแล้ว ตรรกะที่ทำงานกับมันมักอยากจะย้ายเข้ามาอยู่บนออบเจกต์ด้วยเช่นกัน

## เมื่อไรควรใช้ / ข้อแลกเปลี่ยน

แนะนำให้ใช้ parameter object เมื่อค่าชุดเดิมถูกส่งด้วยกันซ้ำ ๆ เมื่อรายการพารามิเตอร์ยาวจนผู้เรียกใช้ส่งลำดับผิด หรือเมื่อคุณรู้สึกได้ว่ามีแนวคิดที่ยังไม่ได้ตั้งชื่อซ่อนอยู่ในรายการอาร์กิวเมนต์ ออบเจกต์บันทึกความสัมพันธ์ไว้และให้ที่อยู่ตามธรรมชาติแก่พฤติกรรมที่เป็นของกลุ่มนั้น

ข้อแลกเปลี่ยนคือมีไทป์เพิ่มที่ต้องนิยาม และมีพิธีรีตองเล็กน้อยในการสร้างมันที่จุดเรียกใช้แต่ละจุด สำหรับการเรียกใช้สองอาร์กิวเมนต์ครั้งเดียวที่ไม่เคยเกิดซ้ำ พิธีรีตองนั้นอาจไม่คุ้มค่า ผลตอบแทนเติบโตขึ้นตามการนำกลับมาใช้: ยิ่งมีฟังก์ชันที่ใช้กลุ่มร่วมกันมากเท่าไร ออบเจกต์ก็ยิ่งคุ้มค่ามากขึ้น — และยิ่งกลายเป็นแม่เหล็กดึงดูดเมท็อดที่เกี่ยวข้อง
