## เจตนา

โค้ดอ่านได้ดีที่สุดเมื่อสิ่งที่อยู่ด้วยกันตั้งอยู่ด้วยกัน Slide Statements ย้าย statement ขึ้นหรือลงเพื่อให้บรรทัดที่เกี่ยวข้องกันกลายเป็นเพื่อนบ้าน — ตัวแปรที่ประกาศไว้ตรงที่มันถูกใช้ครั้งแรก บรรทัด setup ที่อยู่ข้างการเรียกที่มันเตรียมไว้ พฤติกรรมไม่เปลี่ยน เปลี่ยนแค่ลำดับ บ่อยครั้งนี่คือก้าวแรกอันเงียบสงบก่อน Extract Function: คุณเลื่อนชิ้นส่วนที่กระจัดกระจายเข้ามาเป็นบล็อกเดียวที่ต่อเนื่องกัน แล้วยกบล็อกออกไปในการเคลื่อนไหวที่สะอาดเพียงครั้งเดียว

## smell

ตัวแปรถูกประกาศไว้ที่ด้านบนของฟังก์ชัน แต่ไม่ถูกแตะต้องจนกระทั่งสามสิบบรรทัดถัดมา สองบรรทัดที่ทำงานเป็นคู่กันอย่างชัดเจนถูกแยกออกจากกันด้วยโค้ดที่ไม่เกี่ยวข้อง เมื่อคุณพยายาม extract ชิ้นส่วนหนึ่ง คุณพบว่าวัตถุดิบของมันกระจายอยู่ทั่วฟังก์ชัน ดังนั้นการ extract จะต้องใช้พารามิเตอร์ที่เก้กังหรือจะลากเอาบรรทัดที่ไม่ควรอยู่ติดมาด้วย

วิธีรักษาคือเลื่อน statement ที่เกี่ยวข้องกันให้มาอยู่ด้วยกันก่อน เมื่อพวกมันอยู่ติดกันแล้ว โครงสร้างของฟังก์ชันจะชัดเจน และ refactoring ขั้นต่อไปจะง่ายเป็นเรื่องเล็ก

## Before → After

## Mechanics

1. เลือก statement ที่คุณต้องการย้ายและช่องปลายทาง
2. ตรวจโค้ดที่มันเลื่อนผ่านว่ามีการรบกวนกันหรือไม่ การเลื่อนจะปลอดภัยก็ต่อเมื่อลำดับไม่สำคัญระหว่างกัน: statement ที่ย้ายและทุก statement ที่มันผ่านต้องไม่อ่านค่าที่อีกฝ่ายเขียน และทั้งคู่ต้องไม่มี side effect ที่อีกฝ่ายพึ่งพา
3. ย้าย statement ไปยังตำแหน่งใหม่ของมัน
4. **รันเทสต์ของคุณ** ถ้ามีอะไรล้มเหลว แสดงว่าการเลื่อนข้ามการพึ่งพาจริง — ใส่มันกลับและพิจารณาใหม่
5. ทำซ้ำสำหรับ statement ถัดไป โดยเลื่อนทีละอัน เพื่อให้ความล้มเหลวชี้ไปยังการย้ายเพียงครั้งเดียว

## เมื่อใดควรใช้ / ข้อแลกเปลี่ยน

เลื่อน statement เพื่อวางการประกาศไว้ข้างการใช้ครั้งแรกของมัน เพื่อจัดกลุ่มชิ้นส่วนของฟรากเมนต์ที่กำลังจะถูก extract หรือเพียงเพื่อทำให้ setup-และ-action ที่เป็นคู่กันอ่านได้ด้วยกัน มันเป็นการเคลื่อนไหวที่มีความเสี่ยงต่ำและให้ความชัดเจนสูง และเป็นการอุ่นเครื่องที่เหมาะอย่างยิ่งก่อน Extract Function

อันตรายจริงเพียงอย่างเดียวคือการพึ่งพาที่ซ่อนอยู่: บรรทัดที่ดูเหมือนเป็นอิสระแต่แอบพึ่งพา side effect ก่อนหน้า (flag แบบ global, state ที่เปลี่ยนแปลงได้ที่ใช้ร่วมกัน, ลำดับผ่าน I/O) เทสต์ของคุณคือผู้คุ้มกัน เมื่อสงสัย ให้เลื่อนทีละก้าวเล็ก ๆ แล้วรันเทสต์หลังการย้ายแต่ละครั้ง
