## เจตนา

นำชิ้นส่วนหนึ่งของ expression ที่ใหญ่กว่ามากำหนดค่าให้ตัวแปร local ที่มีชื่อดี แล้วใช้ชื่อนั้นแทน sub-expression เดิม รู้จักกันในอีกชื่อว่า Introduce Explaining Variable

## smell

นี่คือวิธีรักษา **cryptic one-liner** — expression ที่หนาแน่นจนคุณต้องแยกแยะมันในหัวทุกครั้งที่อ่าน เงื่อนไข boolean ที่ถักทอจากการเปรียบเทียบสามครั้ง หรือสูตรราคาที่มีเลขคณิตปริศนา ซ่อนความหมายของมันไว้ในเครื่องหมายวรรคตอน การตั้งชื่อแต่ละส่วนเปลี่ยนบรรทัดนั้นให้เป็นสิ่งที่คุณอ่านออกเสียงได้

## Before → After

การตรวจสอบการจัดส่งฝังความหมายของมันไว้ใน expression แบบ boolean เพียงตัวเดียว การตั้งชื่อให้แต่ละส่วนเผยให้เห็นกฎ

## Mechanics

1. ตรวจสอบว่า sub-expression ไม่มี side effect — การ extract อันที่ทำให้ state เปลี่ยนแปลงอาจเปลี่ยนพฤติกรรม
2. ประกาศตัวแปร local แบบ immutable แล้วกำหนดค่า sub-expression ให้มัน
3. ตั้งชื่อตัวแปรที่บอกว่าค่านั้น *หมายความว่าอะไร* ไม่ใช่บอกว่ามันคำนวณอย่างไร
4. แทนที่ sub-expression เดิมด้วยตัวแปรใหม่
5. **รันเทสต์ของคุณ**
6. ถ้า sub-expression เดียวกันปรากฏมากกว่าหนึ่งครั้งในขอบเขต ให้แทนที่ทุกการปรากฏแล้วเทสต์อีกครั้ง

## เมื่อใดควรใช้ / ข้อแลกเปลี่ยน

ใช้มันเพื่อ debug (ตัวแปร local ที่มีชื่อตรวจสอบได้ง่าย) เพื่อแยกสูตรยาว ๆ ออกเป็นชิ้นที่ review ได้ และเพื่อทำให้เงื่อนไขอธิบายตัวเองได้ ภายในฟังก์ชันเดียว ตัวแปร local ก็เพียงพอ เมื่อคำอธิบายเดียวกันมีประโยชน์ทั่วทั้ง object ให้เลือก **Replace Temp with Query** หรือ extract method ออกมาแทน

สิ่งที่ตรงกันข้ามคือ **Inline Variable**: เมื่อชื่อของตัวแปรไม่ได้เพิ่มอะไรเหนือ expression ที่มันถืออยู่ ให้แทนที่ด้วย expression กลับเข้าไปแล้วทิ้งตัวแปร
