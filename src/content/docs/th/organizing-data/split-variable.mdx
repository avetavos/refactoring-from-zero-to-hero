## เจตนา

ตัวแปรหนึ่งควรแทนสิ่งเดียว เมื่อชื่อเดียวถูกกำหนดค่าใหม่ระหว่างทาง จนทีแรกมันถือเส้นรอบรูปและต่อมาถือพื้นที่ มันคือสองตัวแปรที่สวมเสื้อโค้ตตัวเดียวกันจริง ๆ Split Variable ให้แต่ละความหมายที่แตกต่างมีชื่อของตัวเอง ซึ่งถูกกำหนดค่าเพียงครั้งเดียว ผู้อ่านจึงสามารถเชื่อได้ว่าชื่อหนึ่งหมายถึงสิ่งเดียวกันในทุกที่ที่มันปรากฏ และการกำหนดค่าครั้งที่สองที่เคยเปลี่ยนค่ามันก็หายไป

## กลิ่นเหม็น

กลิ่นเหม็นคือ **ตัวแปรที่ถูกกำหนดค่ามากกว่าหนึ่งครั้งสำหรับจุดประสงค์ที่ไม่เกี่ยวข้องกัน** — ไม่ใช่ตัวสะสมที่แท้จริงอย่างตัวนับ loop หรือผลรวมที่กำลังดำเนินอยู่ แต่เป็นชื่อที่ถูกนำกลับมาใช้เป็นช่องเขียนชั่วคราวที่สะดวก เบาะแสสองอย่าง: ตัวแปรนั้นไม่ใช่ตัวแปรเก็บรวบรวม (มันไม่ได้บวกหรือสร้างอะไรข้ามรอบการวน) และความหมายของมันเปลี่ยนไประหว่างการกำหนดค่า เมื่อใดก็ตามที่คุณต้องเลื่อนขึ้นไปแล้วถามว่า "ตอนนี้ค่านี้ถืออะไรอยู่?" ตัวแปรนั้นถูกใช้งานเกินหน้าที่และต้องการถูกแยก

## ก่อน → หลัง

การคำนวณทางฟิสิกส์นำตัวแปร `temp` ตัวเดียวมาใช้ซ้ำ ทีแรกสำหรับเส้นรอบรูป แล้วจึงสำหรับพื้นที่ ตอนหลัง ผลลัพธ์แต่ละอันได้ตัวแปรที่ตั้งชื่ออย่างชัดเจน กำหนดค่าครั้งเดียว เป็นของตัวเอง

## กลไก

1. หาการกำหนดค่าครั้งแรกให้ตัวแปร แล้วเปลี่ยนชื่อมันเป็นชื่อที่อธิบาย *เฉพาะความหมายแรกนั้น* ที่ภาษาอนุญาต ให้ทำมันเป็นค่าคงที่ (`const`, `let` ที่ไม่เปลี่ยนแปลง, `:=` ที่คุณไม่เคยกำหนดค่าใหม่)
2. เปลี่ยนทุกการอ้างอิงระหว่างการกำหนดค่าครั้งแรกและครั้งที่สองให้ใช้ชื่อใหม่
3. **รันเทสต์ของคุณ**
4. ทำซ้ำที่การกำหนดค่าครั้งที่สอง: ประกาศตัวแปรใหม่ด้วยชื่อสำหรับความหมายที่สอง แล้วชี้การอ้างอิงหลังจากนั้นไปยังชื่อใหม่
5. ทำต่อไปจนกระทั่งแต่ละจุดประสงค์ที่แตกต่างเป็นเจ้าของตัวแปรที่แยกกัน กำหนดค่าครั้งเดียว
6. เมื่อตัวแปรที่เปลี่ยนแปลงได้เดิมไม่มีผู้อ่านเหลืออยู่แล้ว ให้ลบมันทิ้ง ใน Rust คุณมักจะตัด `mut` ออกได้ทั้งหมด

## เมื่อใดควรใช้ / ข้อแลกเปลี่ยน

ใช้ Split Variable เมื่อใดก็ตามที่ชื่อหนึ่งถูกกำหนดค่าใหม่เพื่อหมายถึงสิ่งใหม่ — มันกำจัดบั๊กทั้งกลุ่มแบบ "ตอนนี้ค่านี้คืออะไร?" และทำให้แต่ละบรรทัดเข้าใจได้อย่างเป็นอิสระ มันเป็นจุดเริ่มต้นที่เกิดบ่อยของ **Extract Function**: เมื่อแต่ละค่าอยู่ในตัวแปรที่กำหนดค่าครั้งเดียวของตัวเอง ส่วนที่คำนวณมันก็ยกออกได้อย่างสะอาด

ระวังอย่าแยกตัวสะสมที่แท้จริง ตัวนับ loop ผลรวมที่กำลังดำเนินอยู่ หรือตัวสร้างที่เติบโตข้ามรอบการวน *ถูกตั้งใจ* ให้กำหนดค่าใหม่ และการให้แต่ละขั้นมีชื่อของตัวเองจะไร้สาระ จงแยกเฉพาะเมื่อค่าที่ต่อเนื่องกันหมายถึง *สิ่งที่ต่างกัน* ต้นทุนของการแยกคือตัวแปรที่ตั้งชื่อเพิ่มอีกหนึ่งตัว ซึ่งแทบจะถูกกว่าความสับสนจากตัวแปรที่ใช้งานเกินหน้าที่เสมอ
