## Intent

**Hide Delegate**: เมื่อ client เอื้อมผ่านอ็อบเจกต์หนึ่งเพื่อเรียกเมธอดบนอ็อบเจกต์ที่สอง — `person.department.manager` — ให้อ็อบเจกต์แรกมีเมธอดของตัวเองที่ซ่อนการกระโดดนั้น เพื่อให้ client เพียงเรียก `person.manager()` client เลิกพึ่งพิงโครงสร้างที่อยู่เบื้องหลัง `person`

**Remove Middle Man** คือสิ่งตรงข้าม: เมื่ออ็อบเจกต์หนึ่งทำเพียงเล็กน้อยนอกจากส่งต่อการเรียกครั้งแล้วครั้งเล่าไปยัง delegate ของมัน ให้ปล่อยให้ client คุยกับ delegate โดยตรงและลบเมธอดที่ส่งต่อทิ้ง

## The smell

Hide Delegate รักษา **Inappropriate Intimacy** กับภายในของคลาส และเป็นใบหน้าเชิงปฏิบัติของ **Law of Demeter**: คุยกับเพื่อนของคุณ ไม่ใช่เพื่อนของเพื่อนของคุณ สายอย่าง `a.getB().getC().doIt()` ทำให้ผู้เรียก couple กับสองความสัมพันธ์ที่มันไม่มีธุระต้องรู้ smell ที่กลับกันคือ **Middle Man**: คลาสที่เมธอดเกือบทั้งหมดอ่านว่า "return delegate.something()" — ส่งต่อมากเสียจนตัวห่อไม่ได้เพิ่มอะไรเลย

## Before → After

## Mechanics

**Hide Delegate**

1. สำหรับแต่ละเมธอดบน delegate ที่ client เอื้อมไปหา ให้เพิ่มเมธอดที่มอบหมายอย่างเรียบง่ายบน server (อ็อบเจกต์ที่ client ถืออยู่แล้ว)
2. อัปเดต client ให้เรียกเมธอด server ใหม่แทนการเดินไล่ตามสาย
3. **รันเทสต์ของคุณ** หลังแต่ละเมธอดที่คุณเปลี่ยนทิศทาง
4. เมื่อไม่มี client นำทางไปยัง delegate โดยตรง คุณก็สามารถซ่อน accessor ที่เปิดเผยมันได้

**Remove Middle Man** (สิ่งตรงข้าม)

1. เพิ่ม accessor บน server ที่คืนค่า delegate เอง
2. สำหรับแต่ละเมธอดที่มอบหมายซึ่งไม่เพิ่มคุณค่า ให้แทนที่ผู้เรียกของมันด้วยการเรียกโดยตรงผ่าน accessor นั้น
3. **รันเทสต์ของคุณ** ขณะที่คุณทำไป
4. ลบเมธอดที่มอบหมายเมื่อพวกมันไม่มีผู้เรียกเหลืออยู่

## When to use / trade-offs

ซ่อน delegate เมื่อ client เอื้อมผ่านอ็อบเจกต์หนึ่งไปยังเพื่อนบ้านของมัน เมื่อคุณต้องการอิสระในการเปลี่ยนความสัมพันธ์ที่อยู่เบื้องหลังอ็อบเจกต์โดยไม่ทำให้ผู้เรียกทุกตัวพัง หรือเมื่อสายยาวขึ้นจนอ่านเหมือนคำสั่งนำทาง

ความเสี่ยงคือการเลยเถิด: ซ่อนทุก delegate แล้ว server ก็จะเต็มไปด้วยเมธอดที่ส่งต่อจนมัน *กลายเป็น* Middle Man นั่นเองคือจังหวะที่คุณหยิบ **Remove Middle Man** มาใช้และปล่อยให้ client คุยกับ delegate อีกครั้ง การย้ายทั้งสองเป็นปุ่มหมุน — หมุนไปทางการห่อหุ้มเมื่อโครงสร้างผันผวน หมุนไปทางความตรงไปตรงมาเมื่อตัวห่อเป็นเพียงสัญญาณรบกวน
