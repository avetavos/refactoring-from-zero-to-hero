## Intent

ฟังก์ชันที่เริ่มต้นเป็นหน่วยเดียวที่ชัดเจน สามารถเติบโตจนกระทั่งทำสิ่งที่ *ต่างกัน* เล็กน้อยสำหรับผู้เรียกที่ต่างกัน เมื่อส่วนหนึ่งของตัวมันควรแปรผันไปตามผู้เรียก ให้ดึงส่วนนั้นออกจากฟังก์ชันไปยังผู้เรียกแต่ละราย ฟังก์ชันยังคงเก็บแกนกลางที่แชร์กันไว้ ส่วนพฤติกรรมที่แปรผันย้ายไปยังที่ที่ความแตกต่างนั้นอยู่จริง

## The smell

คุณสังเกตว่าตัวเองกำลังจะเอื้อมไปหา boolean parameter — `render(person, includeFooter)` — หรือหวังว่าผู้เรียกรายหนึ่งจะข้ามฟังก์ชันก้อนหนึ่งได้ ความคันนั้นหมายความว่าฟังก์ชันกำลังมัดขั้นตอนที่ไม่ใช่สากลเข้าด้วยกัน ส่วนที่ใช้ร่วมกันและส่วนที่เฉพาะแต่ละผู้เรียกถูกเชื่อมติดกัน นี่คือสิ่งตรงข้ามเป๊ะ ๆ ของ **Move Statements into Function**: ที่นั่น คำสั่งที่เหมือนกันอยากเข้ามา ที่นี่ คำสั่งที่แตกต่างกันอยากออกไป

## Before → After

`renderPerson` ที่ปล่อยบล็อกรูปภาพไว้ท้ายเสมอ ผู้เรียกรายใหม่ต้องการ heading แต่ต้องการ footer ที่ *ต่างออกไป* แทนที่จะเพิ่ม flag เราย้ายบล็อกรูปภาพออกไปยังผู้เรียก

## Mechanics

1. ระบุคำสั่งที่ควรแปรผันไปตามผู้เรียก — มักเป็นส่วนนำหรือส่วนท้ายของฟังก์ชัน ไม่ใช่ตรงกลาง
2. ถ้ามีผู้เรียกเพียงหนึ่งหรือสองราย ให้ copy คำสั่งเข้าไปยังผู้เรียกแต่ละรายโดยตรง ติดกับการเรียก
3. เมื่อมีผู้เรียกจำนวนมาก ให้ทำอย่างปลอดภัย: extract ตัว (ที่แชร์กัน) *ส่วนที่เหลือ* ออกเป็นฟังก์ชันใหม่ ปล่อยให้ฟังก์ชันเดิมเรียกฟังก์ชันใหม่นั้นบวกกับคำสั่งที่แปรผัน แล้วชี้ผู้เรียกไปยังฟังก์ชันที่แชร์กันใหม่ทีละราย
4. **รันเทสต์ของคุณ** หลังต่อสายผู้เรียกแต่ละราย
5. เมื่อผู้เรียกทุกรายเก็บสำเนาคำสั่งที่ย้ายมาของตัวเองแล้ว ให้ลบมันออกจากฟังก์ชันเดิม
6. เปลี่ยนชื่อฟังก์ชันที่แชร์กันถ้าความรับผิดชอบที่แคบลงของมันสมควรได้ชื่อที่ชัดเจนกว่า

## When to use / trade-offs

เอื้อมไปหาสิ่งนี้เมื่อฟังก์ชัน *เกือบ* จะถูกต้องแล้ว แต่มีส่วนหนึ่งที่ต้องต่างกันไปตามผู้เรียก และมิฉะนั้นคุณจะเอื้อมไปหา flag parameter การย้ายความแปรผันออกไปทำให้แต่ละฟังก์ชันโฟกัสที่งานเดียวที่สม่ำเสมอ

ค่าใช้จ่ายคือการทำซ้ำบางส่วนที่จุดเรียก — ผู้เรียกแต่ละรายตอนนี้ทำซ้ำคำสั่งที่ย้ายมา นั่นเป็นที่ยอมรับได้เมื่อผู้เรียกแตกต่างกันจริง ๆ การทำซ้ำนั้นซื่อตรง ถ้าสุดท้ายแล้วคำสั่งกลับเหมือนกันทุกที่ คุณเดินผิดทาง: ใช้ **Move Statements into Function** เพื่อพับมันกลับเข้าไป
