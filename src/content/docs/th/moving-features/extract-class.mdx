## Intent

คลาสหนึ่งค่อย ๆ รับความรับผิดชอบที่สองเข้ามาอย่างเงียบ ๆ ดึงฟิลด์และเมธอดที่รับใช้งานที่สองนั้นออกไปเป็นคลาสใหม่เอี่ยม และให้คลาสเดิมมี reference ไปยังมัน ตอนนี้แต่ละคลาสมีเหตุผลในการเปลี่ยนแปลงที่ชัดเจนเพียงอย่างเดียว

## The smell

นี่คือยารักษา **Large Class** — และเจาะจงกว่านั้นคือคลาสที่มีสองความรับผิดชอบพันกันอยู่ สัญญาณบ่งชี้: กลุ่มย่อยของฟิลด์ที่เปลี่ยนแปลงด้วยกันเสมอ เมธอดที่ทำงานกับกลุ่มย่อยนั้นเท่านั้น หรือชื่อที่ต้องใช้คำว่า "and" เพื่ออธิบายว่าคลาสทำอะไร เมื่อคุณสามารถลากเส้นที่สะอาดผ่านข้อมูลของคลาสได้ ครึ่งหนึ่งในแต่ละด้านก็ต้องการเป็น type ของตัวเอง

## Before → After

`Person` ที่เก็บและจัดรูปแบบหมายเลขโทรศัพท์ด้วย ก่อนหน้านี้ ฟิลด์โทรศัพท์และการจัดรูปแบบโทรศัพท์อาศัยอยู่บน `Person` หลังจากนั้น พวกมันก่อตัวเป็นคลาส `TelephoneNumber` และ `Person` เพียงถือมันไว้หนึ่งตัว

## Mechanics

1. ตัดสินใจว่าจะแยกความรับผิดชอบอย่างไร และสร้างคลาสว่างใหม่สำหรับสิ่งที่คุณกำลังแยกออกมา หากชื่อเดิมไม่เหมาะกับคลาสที่เหลืออีกต่อไป ให้เปลี่ยนชื่อมัน
2. เพิ่มลิงก์จากคลาสเดิมไปยังคลาสใหม่ — โดยทั่วไปเป็นฟิลด์ที่ถือ instance ของคลาสใหม่
3. ย้ายฟิลด์ที่เกี่ยวข้องข้ามไปทีละตัว โดยใช้ **Move Field** และรักษาให้ชุดเทสต์เป็นสีเขียวหลังแต่ละครั้ง
4. ย้ายเมธอดที่อยู่กับฟิลด์เหล่านั้นข้ามไป โดยใช้ **Move Function** เริ่มจากตัวที่อยู่ระดับล่างสุด
5. แทนที่การเข้าถึงฟิลด์โดยตรงของคลาสเดิมด้วยการเรียกผ่าน instance ใหม่
6. **รันเทสต์ของคุณ** หลังการย้ายทุกครั้ง แต่ละขั้นตอนเล็กพอที่แถบสีแดงจะชี้ไปยังการเปลี่ยนแปลงเดียว
7. ทบทวนพื้นผิวสาธารณะ (public surface) ของทั้งสองคลาสและทำให้มันแคบลง — คลาสที่แยกออกมาอาจสามารถซ่อนรายละเอียดที่คลาสเดิมต้องเปิดเผยได้

## When to use / trade-offs

แยกคลาสเมื่อกลุ่มย่อยของฟิลด์และเมธอดก่อตัวเป็นกลุ่มก้อนที่ชัดเจน เมื่อคลาสสรุปได้ยากโดยไม่ใช้คำว่า "and" หรือเมื่อความรับผิดชอบหนึ่งเปลี่ยนแปลงด้วยเหตุผลที่ต่างไปจากส่วนที่เหลือโดยสิ้นเชิง

ต้นทุนคือคลาสที่ต้องตั้งชื่อ สร้าง และนำทางเพิ่มขึ้นอีกหนึ่งคลาส บวกกับชั้นของการมอบหมาย (delegation) หากคลาส *ไม่ได้* ทำสองงานจริง ๆ การแยกออกมาก็เพียงเพิ่มพิธีรีตอง สิ่งตรงข้ามคือ **Inline Class**: หากคลาสใหม่ไม่เคยเติบโตเป็นความรับผิดชอบของตัวเอง ก็พับมันกลับเข้าไป
